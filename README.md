# ⛏️ MineGuard — Smart Mine Safety Monitoring System

> *Because every miner who goes underground deserves to come back home.*

MineGuard is a real-time underground mine safety platform that combines IoT sensor simulation, machine learning, and a live web dashboard to detect hazardous conditions before they become disasters. It monitors toxic gases, seismic activity, and environmental thresholds — and uses trained ML models to classify risk levels and push instant alerts.

## 🧠 Why MineGuard?

Mining is one of the most hazardous occupations on the planet. Toxic gas buildups, oxygen depletion, and seismic instability can turn an ordinary shift into a tragedy within minutes. Traditional monitoring systems are reactive — they alert after danger has already set in.

MineGuard is built to be **proactive**. It continuously streams simulated sensor data, runs it through trained anomaly detection and risk classification models, and flags dangerous trends *before* thresholds are breached. Think of it as a safety co-pilot for mine operators.

## 🔍 What It Monitors

| Sensor | What It Detects | Warning | Danger |
|---|---|---|---|
| **CH₄** (Methane) | Explosion risk | 1.0% vol | 2.5% vol |
| **CO** (Carbon Monoxide) | Toxic gas exposure | 25 PPM | 100 PPM |
| **H₂S** (Hydrogen Sulfide) | Poisonous gas (olfactory fatigue risk) | 1 PPM | 10 PPM |
| **CO₂** (Carbon Dioxide) | O₂ displacement in low areas | 0.5% vol | 1.5% vol |
| **O₂** (Oxygen) | Depletion / breathing risk | 19.5% vol | 16.0% vol |
| **Seismic** | Structural instability / roof collapse | 2.0 Richter | 3.5 Richter |

## 🏗️ System Architecture

```
Sensor Simulation (simulation.py)
        │
        │  MQTT  (mineguard/sensors/#)
        ▼
Flask Backend (app.py)
   ├── Rules Engine       → instant threshold checks → alerts
   ├── AI Inference       → every 30s using trained ML models
   │     ├── IsolationForest    (anomaly detection)
   │     ├── RandomForest       (risk classification: SAFE / WARNING / DANGER)
   │     └── GradientBoosting   (risk probability score 0.0–1.0)
   ├── InfluxDB           → time-series sensor storage
   ├── MongoDB            → alerts, predictions, events
   └── WebSocket (SocketIO) → live dashboard push
        │
        ▼
Live Web Dashboard (mineguard_dashboard_live.html)
```

**Three ML models work together:**
- **Isolation Forest** — unsupervised anomaly detection trained on "normal" safe readings
- **Random Forest Classifier** — classifies each reading as SAFE, WARNING, or DANGER
- **Gradient Boosting Scorer** — outputs a continuous 0.0–1.0 risk probability

## 📁 Project Structure & Folder Setup on Your PC

This is the **exact folder structure** you need to set up locally. Files in the GitHub repo are a bit scattered — here's how to organise them correctly on your machine:

```
mineguard/                          ← Root project folder (create this)
│
├── app.py                          ← Flask backend (main server)
├── simulation.py                   ← Sensor data simulator (MQTT publisher)
├── requirements.txt                ← Python dependencies
├── setup_guide.py                  ← Environment checker & setup instructions
├── mineguard_dashboard_live.html   ← Web dashboard (open in browser)
│
├── models/                         ← ⚠️ CREATE this folder manually
│   ├── isolation_forest.pkl        ← Generated by train_models.py
│   ├── risk_classifier.pkl         ← Generated by train_models.py
│   ├── risk_scorer.pkl             ← Generated by train_models.py
│   ├── training_data.csv           ← Generated by generate_training_data.py
│   └── model_info.json             ← Model metadata
│
└── scripts/                        ← ⚠️ CREATE this folder manually
    ├── generate_training_data.py   ← Step 1: generates training_data.csv
    └── train_models.py             ← Step 2: trains and saves the .pkl models
```

> **⚠️ Important:** The GitHub repo does not have `models/` or `scripts/` folders pre-created. You need to create them yourself and move the files into place as shown above. The `.pkl` model files in the root of the repo belong inside `models/`. The `generate_training_data.py` and `train_models.py` scripts belong inside `scripts/`.

## 🚀 Getting Started

### Prerequisites

Install these once on your system before anything else:

**1. Python 3.8+**
```bash
# Ubuntu / Raspberry Pi
sudo apt install python3 python3-pip

# macOS
brew install python
```

**2. Mosquitto (MQTT Broker)**
```bash
# Ubuntu
sudo apt install mosquitto mosquitto-clients

# macOS
brew install mosquitto
```

**3. InfluxDB 2.x** (time-series database)
```bash
# Ubuntu
wget https://dl.influxdata.com/influxdb/releases/influxdb2_2.7.1_amd64.deb
sudo dpkg -i influxdb2_2.7.1_amd64.deb

# macOS
brew install influxdb
```

**4. MongoDB** (alerts and predictions storage)
```bash
# Ubuntu
sudo apt install mongodb

# macOS
brew install mongodb-community
```

---

### Step-by-Step Setup

#### Step 1 — Clone & Organise Files

```bash
git clone https://github.com/your-username/mineguard.git
cd mineguard

# Create the required folders
mkdir models scripts

# Move files to their correct locations
mv generate_training_data.py scripts/
mv train_models.py scripts/
mv isolation_forest.pkl risk_classifier.pkl risk_scorer.pkl training_data.csv model_info.json models/
```

#### Step 2 — Install Python Dependencies

```bash
pip install -r requirements.txt --break-system-packages
# On Raspberry Pi use pip3
```

#### Step 3 — Configure InfluxDB

```bash
sudo systemctl start influxdb     # Linux
# or: influxd                     # macOS
```

Open your browser at `http://localhost:8086` and:
- Create Organisation: `mineguard`
- Create Bucket: `mineguard`
- Copy your API token → set it as an environment variable:

```bash
export INFLUX_TOKEN=your_token_here
```

#### Step 4 — Start MongoDB

```bash
sudo systemctl start mongod    # Linux
# or: mongod                   # macOS
```

#### Step 5 — Generate Training Data & Train Models

> Skip this step if you already have `.pkl` files in your `models/` folder.

```bash
python scripts/generate_training_data.py
python scripts/train_models.py
```

This creates `training_data.csv` and the three `.pkl` model files in `models/`.

#### Step 6 — Start the MQTT Broker

```bash
mosquitto -v
```

#### Step 7 — Start the Flask Backend

```bash
python app.py
```

You should see:
```
==================================================
  MINEGUARD BACKEND  —  Starting up
==================================================
[MQTT] Connected to broker localhost:1883
[AI]  All models loaded.
```

#### Step 8 — Run the Sensor Simulator

Open a **new terminal** and run:

```bash
python simulation.py
```

This starts publishing realistic simulated sensor readings every 3 seconds.

#### Step 9 — Open the Dashboard

Open `mineguard_dashboard_live.html` directly in your browser. It connects via WebSocket and shows live sensor values, risk levels, and alerts in real time.

## 🤖 ML Models Explained

| Model | Type | Purpose |
|---|---|---|
| `isolation_forest.pkl` | Unsupervised | Detects anomalous/unusual sensor combinations |
| `risk_classifier.pkl` | Random Forest | Classifies reading as SAFE (0), WARNING (1), or DANGER (2) |
| `risk_scorer.pkl` | Gradient Boosting | Outputs a continuous risk score from 0.0 to 1.0 |

The AI runs every **30 seconds** and combines all three model outputs. If the risk score is ≥ 0.75, the system overrides to DANGER regardless of the classifier. Advisories are automatically generated — for example, if H₂S hits 10 PPM, the system warns about olfactory fatigue (you can no longer smell the gas, making it especially dangerous).

## 🛠️ Tech Stack

- **Backend:** Python, Flask, Flask-SocketIO, Flask-CORS
- **Messaging:** MQTT (Paho), Mosquitto broker
- **Databases:** InfluxDB 2.x (time-series), MongoDB (events/alerts)
- **ML:** scikit-learn (IsolationForest, RandomForest, GradientBoosting), joblib
- **Scheduling:** APScheduler
- **Frontend:** Vanilla HTML/CSS/JS with WebSocket (Socket.IO)
